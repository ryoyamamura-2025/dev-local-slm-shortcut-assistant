# AGENTS.md

このファイルは、このリポジトリで作業する Codex とその他のコーディングエージェント向けの指示です。

## プロジェクト概要

- プロジェクト名は **Local Actions**。
- Windows Terminal から受け取った日本語の自然文を、Ollama の Tool Calling で登録済み操作へ変換する個人用ショートカット。
- 現在は Python 3.13、`uv`、Ollama、`qwen3:1.7b` を使用する。
- エントリーポイントは `main.py`。詳しい利用方法、設計判断、既知の制約は `README.md` を参照する。

## 作業開始時

1. `README.md`、`pyproject.toml`、変更対象のコードを読む。
2. `git status --short` で既存の変更を確認し、ユーザーの変更を上書きしない。
3. README の記載だけを根拠にせず、現在のコードと設定を優先する。差異を見つけた場合は必要に応じて両方を整合させる。

## 開発環境とコマンド

- Windows と PowerShell を前提にする。
- Python と依存関係は `uv` で管理する。通常は仮想環境を直接操作せず、`uv run ...` を使う。
- Codex のシェルでは `uv` が `PATH` にない場合がある。`uv` が見つからなければ、`$env:USERPROFILE\AppData\Local\Microsoft\WinGet\Links\uv.exe` を絶対パスで実行する。
- 依存関係の追加・削除には `uv add` / `uv remove` を使い、`pyproject.toml` と `uv.lock` を同期させる。
- 開発実行:

  ```powershell
  uv run main.py <日本語の指示>
  ```

- 現時点では自動テストやリンターは設定されていない。導入していないコマンドを成功条件として扱わない。
- Ollama とモデルが必要な実行確認:

  ```powershell
  ollama list
  uv run main.py 大分駅の近くのカフェを地図で探して
  ```

  この確認はローカルの Ollama サービスを使い、既定ブラウザで新しいタブを開く。ユーザーが実行を求めた場合、または副作用を明示した上で必要な場合にのみ行う。

## 実装方針

- モデルに任せるのは、登録済みツールの選択と引数抽出までとする。
- URL の組み立て、入力検証、実際の操作は決定的な Python コードで実装する。
- 検索語などのユーザー入力は日本語のまま保持し、モデルに不要な翻訳や書き換えをさせない。
- URL クエリは文字列連結で埋め込まず、`urllib.parse.urlencode` などで安全にエンコードする。
- ツールは明示的な許可リストに登録する。モデルが生成した任意の Python、PowerShell、シェルコマンドを実行しない。
- 既存の単純な構成を保ち、小規模な変更のために不要なフレームワークや抽象化を追加しない。
- 公開関数には、目的と引数が分かる簡潔な型注釈と docstring を付ける。
- ソース、ドキュメント、JSON 出力は UTF-8 とし、日本語が文字化けしないことを確認する。

## 変更時の確認

- 変更範囲に応じて、少なくとも構文確認を行う。
- URL 生成やツール関数を変更した場合は、ブラウザを開かずに戻り値を確認できるテストを優先する。
- モデル選択やプロンプトを変更した場合は、Google 検索、Google マップ、X、固定 URL、引数なしツールの代表例を確認する。
- 自動テストを追加した場合は、Ollama やブラウザに依存する処理をモックし、通常のテストがオフラインかつ副作用なしで実行できるようにする。
- 動作、必要環境、コマンド、制約が変わる場合は `README.md` も更新する。

## 完了報告

- 変更した内容、実行した確認、未確認事項を簡潔に報告する。
- Ollama 未起動、モデル未取得、ブラウザ起動を避けたなどの理由で実機確認していない場合は、その点を明記する。
